
# AK-84 Tools

## Free and open source

Набор модулей (скриптов на Python), которые делают работу с данными более комфортной и приятной. Код каждого модуля максимально понятный и лаконичный, используются только широко известные библиотеки. Рассмотрим более подробно каждый модуль.

Модуль query выполняет запрос в реляционную систему управления базами данных, а результаты сохраняет в файл формата CSV. Это избавляет от необходимости вручную повторять запросы в клиентах, а потом вручную экспортировать данные в файлы.

Модуль fetch_items позволяет делать серию последовательных запросов к API. Очень удобен для простого импорта данных, а также для работы с большими языковыми моделями. По сути, простой скрипт для запросов (POST) на нужный API.

Другие модули (hist, plot и scatter) помогают просто и быстро отобразить данные. Например, вы можете посмотреть график показаний датчика или распределение большого массива цен на стиральные машины. Такие вспомогательные обёртки существенно экономят время, если нужно написать код для отображения гистограммы или графика. 

Если вам потребуется быстро и просто выполнить кластерный анализ найденных данных, то для этого есть модуль cluster. Разумеется, речь идёт про быстрый первичный анализ, а не про сложные исследования. Про другие модули вы можете узнать в разделе с примерами.

## Установка AK-84 Tools

Установка не требуется, так как все мои программы поставляются в виде скриптов с открытым исходным кодом. Обратите внимание, что утилиты используют внешние библиотеки. Если у вас не создано виртуальное окружение Python, то это не беда. Выполните следующую команду в пустой директории, куда вы планируете добавить мои скрипты:

```bash
python3 -m venv venv
```

Далее нужно задействовать виртуальное окружение Python:

```bash
source venv/bin/activate
```

А после этого уже можно установить модули для анализа данных:

```bash
pip install numpy pandas matplotlib scikit-learn requests sqlalchemy psycopg2-binary
```

Установка внешних модулей завершена. Приятного использования.

## Примеры

Напомню, что данные скрипты были написаны мной для личных бытовых задач. Я намеренно писал простой и компактный код, который использует только общеизвестные библиотеки (модули). По этой причине у вас не должно возникнуть сложностей с использованием этого кода. Тем более, что скрипты имеют размер всего в несколько строк кода. И это действительно тут случай, когда один взгляд на исходный код заменяет документацию.

Запрос информации у больших языковых моделей:

```python
import fetch_items


# Укажите URL
url = "http://localhost:11434/api/generate"

# Ваш массив объектов (входных данных)
data = [{
    "model": "gemma3:12b",
    "prompt": "Что такое вода? Ответ не больше 256 символов.",
    "stream": False
},
{
    "model": "qwen3:14b",
    "prompt": "Что такое вода? Ответ не больше 256 символов.",
    "stream": False
}]

# Последовательные запросы в один поток
results = fetch_items.fetch_items(url, data)

# Отображение результатов
for result in results:
    print(result["response"]["response"])
    print(result["response_time"])
```

Запрос информации из базы данных и последующее сохранение в файл:

```python
import query


# Ваш запрос в базу данных
SQL = "SELECT pg_size_pretty(pg_relation_size('example')) relation_size;"

# Параметры подключения к базе данных
PARAMS = "postgresql+psycopg2://dev:dev@localhost:5432/dev"

# Исполнение запроса и сохранение результата
query.query(PARAMS, SQL, "result.csv")
```

Визуальное отображение данных:

```python
import hist
import plot
import scatter

# Ваш массив данных, который нужно визуально отобразить
```
